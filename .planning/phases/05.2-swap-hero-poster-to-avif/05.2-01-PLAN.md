# Phase 05.2 Plan 01: AVIF poster swap

## Goal

Close the mobile LCP bottleneck by replacing the 2.55 MB raw PNG hero poster with a 112 KB AVIF asset. The `<video poster>` path bypasses Next/Image entirely, so the only way to fix that fetch is to make the static file at the URL small.

## Tasks

### Task 1: Encode + wire + document

**Sub-steps:**

1. Encode the AVIF via ffmpeg libsvtav1: `ffmpeg -i public/hero/homepage-hero-start.png -c:v libsvtav1 -crf 30 -preset 5 -frames:v 1 public/hero/homepage-hero-start.avif`. Verify size < 200 KB.
2. Land `scripts/encode-hero-poster.sh`: extracts the first frame from `homepage-hero-720p.mp4`, re-encodes, refuses to overwrite if the output exceeds the 200 KB budget. Chmod +x.
3. Update `src/content/homepage-hero.ts:55`: `startFrame` path changes from `.png` to `.avif`, comment block refreshed to document both consumers (Image + video poster).
4. Delete `public/hero/homepage-hero-start.png` (2.55 MB; regenerable via the new script).
5. Update `scripts/check-hero-assets.sh` comment to reflect the post-5.2 state (no PNG byte budget needed; AVIF budget is enforced by `tests/hero/poster-avif-negotiation.spec.ts`).
6. Add `/.lighthouseci/` to `.gitignore`.
7. Update `.planning/ROADMAP.md` Phase 5.2 block: populate Plan 01 entry, drop the "TBD" placeholder.
8. Update `.planning/STATE.md`: roadmap evolution entry for 5.2 ship, current position pointer to 05.1 Plan 03 Task 1 resume.
9. Update `HANDOFF.md`: brief note in the Hero performance section about the poster swap.
10. Typecheck (`npm run typecheck`), lint (`npm run lint`), run the three hero specs locally (`tests/hero/poster-avif-negotiation.spec.ts`, `tests/hero/source-ladder.spec.ts`, `tests/responsive/hero-mobile-video-free.spec.ts`) to confirm zero regressions.
11. One atomic commit `feat(05.2): swap hero poster to AVIF to close HERO-04 LCP gate` with all of the above.
12. Push to `phase-05.1` branch to trigger Vercel preview build.

## Pitfalls

- **Forgetting to delete the PNG.** If left in `public/hero/`, the asset size budget guard could be extended in a later phase to cover it, but for now it's just disk weight and confusion. Delete it.
- **Editing `tests/hero/poster-avif-negotiation.spec.ts`.** Do NOT. The spec's substring match (`url.includes("homepage-hero-start")`) covers both `.png` and `.avif` source paths; it asserts the `/_next/image` response is AVIF + under 200 KB regardless of source extension. The HERO-03 contract is unchanged.
- **CRF that's too low.** CRF below 25 starts breaking the 200 KB budget on this image. The encode script's hard ceiling at 200 KB catches the regression on a future re-run.
- **Touching `HomepageHero.tsx`.** No code change is needed. Both consumers reference `heroCinematic.media.startFrame`, which is repointed at the data layer. Resist any urge to refactor.
