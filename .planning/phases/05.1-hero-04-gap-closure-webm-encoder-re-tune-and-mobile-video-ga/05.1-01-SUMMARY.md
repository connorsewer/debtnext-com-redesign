# Phase 05.1 Plan 01: Strip WebM ladder + tighten mobile gate — Summary

Drop the 3 WebM binaries and the libvpx-vp9 build path, then narrow the `<source media>` algebra so phones under 768px match zero video sources. Combined with the existing `!isMobile` React gate, this makes the mobile-video-free invariant structural (CSS algebra) rather than only React-conditional.

## Phone Render Contract (post-fix)

When a 412×823 phone hits `/`:

1. SSR HTML emits the `<video>` element (Mechanism 1 from RESEARCH §Defect A unchanged) because `isMobile` defaults to `false` server-side.
2. Browser parses the HTML and walks the 3 `<source>` children:
   - 360p: `(min-width: 768px) and (max-width: 1023px)` — fails (412 < 768)
   - 540p: `(min-width: 768px) and (max-width: 1439px)` — fails (412 < 768)
   - 720p: unbounded — but already after both predecessors, and the source-selection algorithm has nothing to fall back to since no media query matched
3. `video.networkState === NETWORK_NO_SOURCE`. No video bytes are fetched.
4. React hydrates, `useEffect` runs `matchMedia("(max-width: 767px)")` → true → `setIsMobile(true)` → re-render removes the `<video>` element entirely.

Plan 05.1-02 will add the Playwright network-watcher regression net that asserts zero `/\.(mp4|webm|m4v|mov)$/` requests fire during the full page lifecycle at 412×823.
