## Phone Hero Fallback Contract

This is the **single net-new visual decision** in Phase 05.1. After D-04's `<source media>` tightening, a viewport &lt;768px matches zero video sources. This section locks what the phone renders instead.

### Render layers at 412×823 (post-fix)

| Layer | Element | Source | Contract |
|-------|---------|--------|----------|
| 1 (LCP target) | `<Image>` AVIF | `/_next/image?url=%2Fhero%2Fhomepage-hero-start.png&w=...&q=...` (negotiated to AVIF when `Accept: image/avif` present, else WebP, else PNG) | Sub-200 KB AVIF at 1920w (HERO-03 contract, verified 150,193 B at w=1920). `<Image preload fetchPriority="high">` at `HomepageHero.tsx` lines 170-181. `object-cover`. Fills the `h-screen` (`100vh`) container, anchored top-left, scales to cover both axes. |
| 2 (video element during SSR-to-hydration window) | `<video>` element | `HomepageHero.tsx` lines 191-209 | Present in SSR HTML (because `isMobile` defaults to `false` server-side). Zero `<source>` children match at &lt;768px after D-04 → `video.networkState === NETWORK_NO_SOURCE` → zero bytes fetched. Element painted at `opacity: 0` (line 198), so invisible. Removed entirely after React hydration when `setIsMobile(true)` re-renders without it (`!isMobile` gate at line 190). |
| 3 (vignette) | Decorative gradient | `bg-gradient-to-b from-black/35 via-transparent to-black/40` | Unchanged from desktop. Handles nav + disclaimer legibility on top of the AVIF. |
| 4 (overlay copy) | Headline + subhead + form + disclaimer pill | `HomepageHero.tsx` lines 248-300 | Unchanged. Renders identically on phones and desktop. |

### Locked phone-render contract

| Property | Value | Rationale |
|----------|-------|-----------|
| LCP element | AVIF poster painted by `<Image preload fetchPriority="high">` | HERO-03 contract; AVIF is sub-200 KB; same element owns LCP on every viewport per `.impeccable.md` §Performance |
| Visual stand-in for the video | The AVIF poster, statically rendered (no motion, no scrub) | Phones intentionally do not run the GSAP scrub cinematic (`useGSAP` line 61 bails when `isMobile === true`). The static start-frame is the cinematic's representative still. |
| Image source chain (browser content-negotiation order) | AVIF → WebP → PNG | `next.config.ts` `images.formats: ['image/avif', 'image/webp']` (HERO-03 wave 0). PNG is the source-of-truth fallback; raw PNG is 2.55 MB but is NEVER served on the LCP path (Next/Image transforms it). |
| `alt` attribute | `alt=""` with `aria-hidden="true"` | Decorative atmospheric image per `DESIGN.md` §7.2 ("decorative media uses empty alt text or `aria-hidden='true'`"). The hero `<h1>` provides the accessible meaning of the section. |
| Vertical reserved height | `h-screen` (`100vh`) — full viewport | Inherits the desktop sticky pin container (`HomepageHero.tsx` line 166). The cinematic's `~260vh` of GSAP pin is desktop-only and does not apply on phones (the `useGSAP` bail at line 61 means no ScrollTrigger is created, so no `.pin-spacer` injection). Phones see a normal 100vh hero followed immediately by the Platform section. |
| Aspect ratio behavior | `object-cover` fills both axes, anchored top-left | AVIF source is 16:9 landscape (1920×1080 master). On a 412×823 portrait viewport, `object-cover` crops the horizontal edges to fill the height. Verified in the existing `<Image fill sizes="100vw">` setup. |
| Reduced-motion behavior on phones | n/a — no motion exists on phones in the first place | Phones never run the GSAP scrub. `prefers-reduced-motion: reduce` therefore has nothing to suppress on phones. Contract is the same as the desktop reduced-motion contract: static AVIF + overlay copy. |
| Reduced-motion behavior on desktop | `motion-reduce:hidden` on the `<video>` (line 199) + `useGSAP` bail at line 62 → static start-frame PNG (AVIF via `<Image>`) owns the viewport; overlay + form shown statically | Existing contract from Phase 5; carries forward unchanged. |

### What phones do NOT render

- No `<video>` element after hydration (gated by `!isMobile` at line 190).
- No `.mp4`, `.webm`, `.m4v`, or `.mov` network request fires during navigation, post-load, or scroll. **D-06 Playwright spec is the regression net for this invariant.**
- No GSAP ScrollTrigger pin (no `.pin-spacer` injection); no scrub binding; no framed-dashboard finale crossfade. Phones go straight from the static hero to the Platform handoff section.

### Surfaces that have no visual contract (explicit note for the checker)

Two Phase 05.1 deliverables intentionally do not have a visual contract:

| Surface | Why no visual contract |
|---------|-----------------------|
| `tests/responsive/hero-mobile-video-free.spec.ts` (D-06) | Playwright network-request assertion. Headless test artifact; never rendered to a user. No design surface. |
| `scripts/check-hero-assets.sh` (D-07) + `.github/workflows/perf.yml` wiring | Shell-script size-budget guard executed in CI. Console output only; never rendered to a user. No design surface. |

These pass the design-contract check by virtue of having no design surface to contract. The checker should not flag absent visual decisions here.
